    CentOS7 配置NTP¶

    一、NTP概念¶

    网络时间协议（NTP, Network Time Protocol）是一种用于在计算机网络中同步时间的协议。NTP 允许计算机和其他网络设备保持一致的时间，即使在不同的地理位置和不同的网络环境中也能做到时间同步。NTP 主要用于确保分布式系统中的所有设备和服务都有统一的时间，以便于日志记录、事件排序、定时任务等。

    二、NTP服务工具¶

    ntpdate、ntp 和 chrony 是三个用于时间同步的工具和服务，每个工具都有其特定的用途和特点。以下是对这三种工具的详细比较和说明：

    1. ntpdate：
    2. 功能：ntpdate 是一个用于手动同步系统时间的工具。它会向指定的 NTP 服务器发送时间请求，并根据返回的时间更新系统时钟。
    3. 特点：
      • 一次性同步：ntpdate 仅执行一次时间同步操作，并不会持续运行。它适用于需要临时更新系统时间的场景。
      • 不适合持续同步：不适合需要持续时间同步的环境，因为它不会持续运行并监控时间偏差。
    4. 注意：ntpdate 已被认为是过时的工具，现代系统通常推荐使用 chrony 或 ntpd。
    5. ntp：
    6. 功能：ntp 是一个守护进程 (ntpd)，负责持续地同步系统时间。它会定期与 NTP 服务器通信，自动调整系统时间，以保持准确性。
    7. 特点：
      • 持续运行：ntpd 是一个守护进程，持续运行并维护系统时间同步。
      • 复杂配置：配置较为复杂，支持多种同步模式和高级特性。
      • 自动调整：会根据网络延迟等因素自动调整时间，以提高同步精度。
    8. 注意：ntpd 的时间同步速度较慢，但稳定性和精度较高，适合长期运行的服务器。
    9. chrony：
    10. 功能：chrony 是一个现代的时间同步工具，旨在取代传统的 NTP 工具。它包括两个主要组件：chronyd（服务守护进程）和 chronyc（命令行客户端）。
    11. 特点：
      • 快速同步：chrony 提供更快的时间同步速度，尤其在虚拟环境和不稳定的网络条件下表现优越。
      • 高效：适合经常处于变化的网络环境或频繁重启的系统。
      • 精度高：提供更精确的时间同步服务，支持更广泛的环境和应用场景。
    12. 注意：chrony 提供的功能更加现代化和高效，是许多系统管理员推荐的时间同步解决方案。

    下面分别介绍三种工具的安装和使用。

    三、常用的NTP服务器¶

    • Google NTP 服务器：

    • time.google.com；

    • Google 提供的 NTP 服务，具有较高的可靠性和全球分布的服务器。

    • NTP Pool Project 服务器：

    • pool.ntp.org；

    • 这个项目提供了全球分布的 NTP 服务器集群，按地域可细分，如 cn.pool.ntp.org 专用于中国地区。

    • 阿里云 NTP 服务器：

    • ntp.aliyun.com；

    • ntp1.aliyun.com；

    • 阿里云提供的 NTP 服务，在中国境内延迟较低，适合国内用户使用。

    • 华为云 NTP 服务器：

    • ntp1.cloud.huawei.com；

    • ntp2.cloud.huawei.com；

    • ntp3.cloud.huawei.com；

    • 华为云提供的 NTP 服务，具有高可靠性。

    • 国家授时中心 NTP 服务器：

    • ntp.ntsc.ac.cn；

    • 中国科学院国家授时中心提供的 NTP 服务，专门为中国境内用户提供高精度的时间同步。

    • 腾讯云 NTP 服务器：

    • ntp.tencent.com；

    • 腾讯云提供的 NTP 服务，适用于国内用户。

    • Microsoft NTP 服务器：

    • time.windows.com

    • Microsoft 提供的 NTP 服务，适合 Windows 用户。

    四、安装并使用ntpdate工具¶

    1.修改系统时间¶

    这里我们先修改系统时间，使用 data 命令随便修改为一个时间：

    Bash
    # 查看当前时间
date
# 显示当前时间
Mon Aug 19 22:01:01 CST 2024

# 修改当前时间为2024年8月18日15:30
date -s "2024-08-18 15:30:00"
# 再次查看当前时间
date
# 显示当前时间
Sun Aug 18 15:30:03 CST 2024

    也可以使用时间戳来修改：

    Bash
    # 查看当前时间戳
date +%s
# 显示当前时间戳
1724075698

# 修改当前时间戳
date 112523092020.18
# 再次查看当前时间
date
# 显示当前时间
2020年 11月 25日 星期三 23:09:18 CST

    2.安装ntpdate工具¶

    使用 yum 命令进行安装：

    Bash
    yum install -y ntpdate

    3.使用ntpdate工具¶

    使用 ntpdate 命令更新系统时间：

    Bash
    ntpdate ntp.aliyun.com
# 显示信息
19 Aug 22:05:44 ntpdate[8026]: step time server 203.107.6.88 offset 109868.479535 sec

    这里使用的是阿里云的 NTP 服务器，在中国境内延迟较低，适合国内用户使用。

    再次通过 date 命令查看当前时间，发现已正常：

    Bash
    date
# 显示当前时间
Mon Aug 19 22:06:10 CST 2024

    使用 hwclock 命令显示系统硬件时钟的当前时间：

    Bash
    hwclock --show
# 或者更简洁的版本
hwclock -r

# 显示信息
Mon 19 Aug 2024 10:08:08 PM CST  -0.428151 seconds

    -0.428151 seconds：表示硬件时钟与系统时钟之间的时间差，单位为秒，这里表示硬件时钟比系统时钟慢约 0.428 秒。

    可以使用以下命令将系统时钟的时间同步到硬件时钟：

    Bash
    hwclock --systohc

    或者将硬件时钟的时间同步到系统时钟：

    Bash
    hwclock --hctosys

    可以使用 timedatectl 命令设置时区，比如设置为中国标准时间（CST，Asia/Shanghai 时区）：

    Bash
    timedatectl set-timezone Asia/Shanghai

    可以设置定时任务，每天在中午12点自动运行 ntpdate 命令，并从 ntp.aliyun.com 服务器同步系统时间：

    Bash
    # 配置定时任务
crontab -e
# 添加以下内容
00 12 * * * /sbin/ntpdate ntp.aliyun.com

    五、安装并使用ntp工具¶

    1.配置ntp服务端¶

    1.1 安装ntp工具¶

    使用 yum 命令进行安装：

    Bash
    yum install -y ntp

    1.2 修改ntp配置文件¶

    编辑 ntp的配置文件 /etc/ntp.conf，配置以下内容：

    Bash
    # 配置 NTP 服务器的上游时间源
server ntp.aliyun.com iburst
server ntp1.aliyun.com iburst
# 允许特定 IP 范围的客户端访问
restrict 10.22.51.0 mask 255.255.255.0 nomodify notrap

    命令参数解释说明：

    • iburst：用于加速 NTP 客户端与服务器之间的时间同步过程，特别是在初始同步时。

    • restrict ：该指令用于配置 NTP 服务器的访问控制策略，以控制哪些客户端可以与 NTP 服务器进行通信，以及可以执行的操作。

    • nomodify：这个选项禁止客户端修改 NTP 服务器的配置，客户端不能对 NTP 服务器进行配置更改。

    • notrap：这个选项禁止客户端使用 NTP 服务器的 “trap” 功能，NTP 服务器不会为这个客户端提供对特定事件的通知。

    1.3 启动ntpd服务¶

    Bash
    # 启动ntpd服务
systemctl start ntpd
# 查看服务状态
systemctl status ntpd
# 设置服务开机自启
systemctl enable ntpd

    1.4 检查时间同步状态¶

    执行以下命令查看时间源列表：

    Bash
    ntpq -p

    显示信息如下：

    Bash
         remote           refid      st t when poll reach   delay   offset  jitter
==============================================================================
*203.107.6.88    100.107.25.114   2 u    5  128  377   48.779   -8.512   6.644
+120.25.115.20   10.137.53.7      2 u   14  128   71    6.127  -11.582   7.182

    2.配置ntp客户端¶

    2.1 安装ntp工具¶

    使用 yum 命令进行安装：

    Bash
    yum install -y ntp

    2.2 修改ntp配置文件¶

    编辑 ntp的配置文件 /etc/ntp.conf，配置以下内容：

    Bash
    # 设置 NTP 服务器的地址
server 10.22.51.49 iburst

    2.3 启动ntpd服务¶

    Bash
    # 启动ntpd服务
systemctl start ntpd
# 查看服务状态
systemctl status ntpd
# 设置服务开机自启
systemctl enable ntpd

    3.验证配置¶

    在客户端上，使用以下命令来检查 NTP 服务状态和同步情况：

    Bash
    ntpq -p

    执行后显示的信息如下：

    Bash
         remote           refid      st t when poll reach   delay   offset  jitter
==============================================================================
*10.22.51.49     203.107.6.88     3 u   17   64    1    0.542    0.073   0.077

    显示 * 号，说明已经同步成功。

    输出字段解释：

    • remote：显示当前 NTP 客户端正在使用的主服务器的地址，前面的 * 表示这是被选为主时间源的服务器。
    • refid：远程服务器的参考 ID，指示它从哪个服务器或时间源获得时间。
    • st：显示服务器的层级，层级越小表示距离准确时间源越近。例如，1 表示直接与外部时间源同步，2 表示与一个层级为 1 的服务器同步。这里 3 表示这是第三层服务器（即它是与第二层服务器同步的）。
    • t：表示时间同步的类型，常见的有 u（UDP）或 l（本地）等。这里是 u（UDP），通常指使用 UDP 协议进行同步。
    • when：显示上次与远程服务器进行通信的时间，单位是秒。这里表示上次与这个服务器进行通信的时间为 17 秒前。
    • poll：显示客户端与服务器之间的请求间隔，单位是秒，表示客户端多久会发送一个请求。这里表示请求间隔为 64 秒。这个间隔是动态调整的，用于平衡时间同步的频率和网络负载。
    • reach：显示到达服务器的响应状态，通常是一个八位的八进制数，每一位表示一个请求是否成功接收到响应。这里 1 表示最近的一次请求成功得到了响应。377 代表之前有多个请求的响应状态，1 表示最后一次请求成功，其他七位都是 1。
    • delay：显示客户端与服务器之间的往返延迟时间，单位是毫秒。
    • offset：显示客户端与服务器之间的时间偏差，单位是毫秒。这个值表明客户端时间与服务器时间之间的差异。
    • jitter：显示时间同步的抖动，即时间偏差的波动，单位是毫秒。

    总结：从这个输出结果来看， NTP 客户端与主服务器 10.22.51.49 正在正常同步，且主服务器的时间源被选中作为同步源。延迟和偏差都很小，说明同步状态良好。reach 为 1 表示最近的请求成功得到了响应，这表明连接稳定。

    4.测试客户端与服务端的时间同步情况¶

    4.1 临时修改服务端时间¶

    使用 date 命令临时修改服务端的系统时间：

    Bash
    date -s "2024-08-19 12:00:00"

    确认时间已被修改：

    Bash
    date

    4.2 验证时间同步情况¶

    在客户端上重启ntpd服务：

    Bash
    systemctl restart ntpd

    再次查看客户端的系统时间，发现时间已成功同步：

    Bash
    date

    六、安装并使用chrony工具¶

    1.配置chrony服务端¶

    1.1 安装chrony工具¶

    使用 yum 命令进行安装：

    Bash
    yum install -y chrony

    1.2 修改chrony配置文件¶

    编辑 chrony的配置文件 /etc/chrony.conf，配置以下内容：

    Bash
    # 配置 NTP 服务器的上游时间源
server ntp.aliyun.com iburst
server ntp1.aliyun.com iburst
# 允许特定 IP 范围的客户端访问
allow 10.22.51.0/24
# 开启手动校准时间功能，指定本地时钟作为时间源
local stratum 10

    命令参数解释说明：

    • local stratum 10：是在 NTP 配置中用于设置本地时间源的层级（stratum）的一条命令。它通常用于指定本地时钟的层级，stratum 表示时间源的层级或距离主时间源的距离。
    • local：表示使用本地计算机的时钟作为时间源。
    • stratum 10：设置本地时钟的 stratum 层级为 10。这个值通常设置为较高的层级，表示本地时钟是一个备用时间源，而不是主要时间源。

    1.3 启动chronyd服务¶

    Bash
    # 启动chronyd服务
systemctl start chronyd
# 查看服务状态
systemctl status chronyd
# 设置服务开机自启
systemctl enable chronyd

    1.4 检查时间同步状态¶

    执行以下命令查看时间源列表：

    Bash
    chronyc sources

    显示信息如下：

    Bash
    MS Name/IP address         Stratum Poll Reach LastRx Last sample               
===============================================================================
^- 203.107.6.88                  2   6    77    12  -1920us[-2251us] +/-   31ms
^* 120.25.115.20                 2   6   157     7   -104us[ -439us] +/- 4163us

    2.配置chrony客户端¶

    2.1 安装chrony工具¶

    使用 yum 命令进行安装：

    Bash
    yum install -y chrony

    2.2 修改chrony配置文件¶

    编辑 chrony的配置文件 /etc/chrony.conf，配置以下内容：

    Bash
    # 设置 NTP 服务器的地址
server 10.22.51.63 iburst

    2.3 启动chronyd服务¶

    Bash
    # 启动chronyd服务
systemctl start chronyd
# 查看服务状态
systemctl status chronyd
# 设置服务开机自启
systemctl enable chronyd

    3.验证配置¶

    在客户端上，使用以下命令来检查 Chrony 服务状态和同步情况：

    Bash
    chronyc sources

    执行后显示的信息如下：

    Bash
    210 Number of sources = 1
MS Name/IP address         Stratum Poll Reach LastRx Last sample               
===============================================================================
^* 10.22.51.63                   3   6    37    54   +371ns[+1304us] +/-   31ms

    显示 * 号，说明已经同步成功。

    输出字段解释：

    • ^\*：^* 表示当前被选择的主要时间源。* 表示这是当前使用的主要时间源，^ 表示它被选中，但 chrony 可能正在评估其他时间源。
    • *：表示当前同步的源。
    • +：表示可选的时间源。
    • -：表示被组合算法排除的时间源。
    • ?：表示连接已丢失或数据包未通过测试的时间源。
    • x：表示虚假的时钟源，即它的时间与大多数其他源不一致。
    • ~：表示该时间源有太多可变性的来源。
    • 10.22.51.63：时间源的 IP 地址。当前的主要时间源是 10.22.51.63。
    • 3：时间源的 stratum 层级。层级为 3 表示它是从 stratum 2 时间源获取时间的。
    • 6：轮询间隔（单位是秒），6 表示 chrony 每隔 6 秒向该时间源请求数据。
    • 37：reach 值，以八进制表示。37 表示 chrony 已经成功地与时间源进行了 37 次通信。
    • 54：LastRx，上次接收数据的时间间隔（单位是秒），54 表示 54 秒前接收到数据。
    • +371ns[+1304us]：Last sample 部分显示了上次接收到数据时的偏差和抖动：
    • +371ns 表示从时间源的时间偏差为 +371 纳秒。
    • [+1304us] 表示时间源的时间与客户端时间之间的偏差为 +1304 微秒。
    • +/- 31ms：jitter，表示测量的抖动或波动，31ms 表示时间测量过程中出现的抖动范围为 ±31 毫秒。

    4.测试客户端与服务端的时间同步情况¶

    4.1 临时修改服务端时间¶

    使用 date 命令临时修改服务端的系统时间：

    Bash
    date -s "2024-08-19 12:00:00"

    确认时间已被修改：

    Bash
    date

    4.2 验证时间同步情况¶

    在客户端上重启chronyd服务：

    Bash
    systemctl restart chronyd

    再次查看客户端的系统时间，发现时间已成功同步：

    Bash
    date

    此时查看时间同步状态：

    Bash
    chronyc sources

    显示信息如下：

    Bash
    210 Number of sources = 1
MS Name/IP address         Stratum Poll Reach LastRx Last sample               
===============================================================================
^* 10.22.51.63                  10   6    17    36  -4183ns[  -26us] +/-   76us

    可发现该服务器的层数显示为第 10 层，表示时间源不可靠，即手动授予时间。

    执行以下命令，在服务端上可查看哪些客户端正在同步时间：

    Bash
    chronyc clients

    显示信息如下：

    Bash
    Hostname                      NTP   Drop Int IntL Last     Cmd   Drop Int  Last
===============================================================================
10.22.51.65                    10      0   7   -    10       0      0   -     -

    七、总结¶

    建议只需要在一台服务器上配置ntp或chrony工具，然后其他服务器使用ntpdate工具手动去指向该服务器来同步时间即可。
